## ЁЯза Deep Tech Stack & Concepts

- **DOM Selection & Mutation** тАФ `document.getElementById()`, `innerHTML`, `style.background/backgroundColor`, `style.display`, `.remove()`, `classList.contains()` тАФ used pervasively for direct element manipulation without jQuery or frameworks.
- **Inline Event Binding** тАФ `onclick`, `onchange`, `onblur`, `oninput`, `onmouseenter`, `onmouseleave`, `onscroll` attributes attached directly to HTML elements, dispatching both single-purpose and multi-branch handler functions.
- **Type Coercion & Parsing** тАФ `parseInt()` for integer addition, `parseFloat()` for floating-point decimal operations, and `eval()` in the Homework 2 calculator for arbitrary arithmetic string resolution (with `replaceAll()` to sanitize `├Ч`/`├╖` to `*`/`/`).
- **Array State Management** тАФ Parallel arrays (`students[]`, `genders[]`, `averages[]`) as a pre-object-relational data pattern; `push()` for insertion, `splice(i,1)` for index-based deletion, and `filter()` (commented out as alternative). Reverse-loop iteration (`for i=length-1 down to 0`) solved index-shifting bugs during multi-delete.
- **String Normalization for Search** тАФ `.toUpperCase()` + `.trim()` on both query and target to enable case-insensitive matching with tolerance for surrounding whitespace.
- **Game State Machine** тАФ Turn-based boolean flags (`turn`, `end_game`, `game_started`) and two-card memory slot pattern (`first_card`, `second_card`) to gate valid input, prevent double-clicks, and sequence reveal/hide logic.
- **Asynchronous APIs** тАФ `setInterval()` / `setTimeout()` / `clearInterval()` for timer bars, staggered card-hide delays, and auto-triggered callbacks with state re-entry (`game_started = true` after `hide_numbers`).
- **Shuffle Algorithm (Fisher-Yates)** тАФ In-place array shuffle with descending index and destructuring swap (`[a[i], a[j]] = [a[j], a[i]]`), applied to `game_numbers` for randomized card placement each session.
- **CSS Custom Properties (Theming)** тАФ `:root` variables with `document.querySelector(':root').style.setProperty()` to toggle dark/light mode by rewriting `--color1`, `--color2`, `--color3`, and `--text_color` at runtime.
- **CSS Layout Techniques** тАФ `display: flex` for container row alignment, `grid-template-columns: repeat(2, 1fr)` for product grids, `grid-gap`, `position: absolute` for overlay elements, `overflow-y: scroll` for scrollable cart, `border-radius` with `inset` box-shadows for neumorphic calculator styling.
- **Dynamic Template Literals** тАФ Backtick-delimited HTML strings (`<div id='stu_${i}'...>`) for rendering dynamic data rows with inline IDs for direct element targeting.
- **Font Awesome Integration** тАФ Icon-based UI elements (`fa-cart-shopping`, `fa-trash`, `fa-sun`, `fa-moon`) with state-driven class swaps for theme mode indication.

## ЁЯТб Problem-Solving Case Study

The most instructive challenge came from the **Memory Game** progression across Sessions 8 and 9. In Session 8, the card values were hardcoded and the game had no randomization or timer тАФ a simple click-to-reveal system with a 200ms `setTimeout` fallback on mismatch. The glaring problem: cards were predictable and the player had no pressure mechanism. In Session 9, I solved both issues. First, I introduced a **Fisher-Yates shuffle** on the `game_numbers[]` array at load time, ensuring every round had a unique card layout. Second, I built a **visual countdown timer** using `setInterval(time_step, 50)` that decremented a pixel-width variable (`timer_width -= 1`), progressively shrinking a `div` from 495px to 0 тАФ creating an intuitive, real-time pressure bar. I also fixed the **same-card double-click bug** by adding an early-return guard (`if (first_card == second_card) { second_card = 0; return; }`) and gated all gameplay behind a `game_started` boolean toggled by the "Start Game" button.

Another critical breakthrough was in the **array delete functions**. Initially, a forward-iterating loop would skip elements after a `splice()` call because indices shifted left. The fix was reversing the loop direction (`for var i = students.length - 1; i >= 0; i--`) so deletions only affected already-processed indices. This pattern was then reused for gender-based filtering (`delete_by_gender()`) and threshold-based pruning (`delete_below_average()`), turning a buggy single-delete into a robust batch-delete system.

The **Homework 2 calculator** also showcased a design leap: it replaced the per-button operation logic of Sessions 2 and 3 with a flat `eval()` strategy, using `res.value += char` for input accumulation and `replaceAll()` to convert display symbols back to JS operators before evaluation. Combined with a dark/light theme toggle via `data-theme` attribute and CSS custom properties, it was a clear leap in code minimalism and UX polish.
